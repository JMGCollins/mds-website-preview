Current pricing model

Pricing built around system depth and phrase volume.

MDS pricing is intentionally product-sensitive. Specialty documentation systems may vary in price based on specialty, depth, and intricacy, while the generator is best priced by the number of dot phrases created rather than by time-limited access.

For now, DDS has a defined three-tier package structure, FMDS is shown with a single placeholder package, and the generator follows a phrase-count model.

Locked generator principle

Phrase-count model

  • Each new generated dot phrase counts as one usage.
  • Refinement passes do not count toward usage.
  • Higher-volume packages should have a lower effective per-phrase cost.

Important usage rules

These rules make the model understandable and fair.

New phrase

A newly generated dot phrase counts as one usage.

Batch mode

Each individual phrase in a batch still counts as a single usage.

Refinement

Post-generation refinements such as formatting or signature block requests do not count as additional usage.
