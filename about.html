About the platform

Medical Documentation System was built from the belief that documentation should function like a system.

MDS was created by Joshua Collins, DO, a flight surgeon and dermatology resident, and is being developed as a physician-built outpatient documentation platform focused on structure, consistency, and the long-term value of organized clinical language.

The underlying philosophy is straightforward: better documentation is not only about writing faster. It is about creating systems that are reusable, coherent, clinically defensible, and capable of supporting future AI-assisted workflows responsibly.

Founder

Joshua Collins, DO

  • Flight surgeon and dermatology resident building MDS from firsthand experience with real-world clinical documentation demands.
  • Focused on outpatient documentation quality, structure, and clinically usable workflow design.
  • Developing specialty systems and generator logic with a professional, physician-facing standard in mind.

Core philosophy

The platform is built around documentation principles that can scale across products and specialties.

Clarity

Clean structure matters

Clinical documentation becomes stronger when it is organized, predictable, and intentionally formatted.

Consistency

Systemization creates value

When families, wording, placeholders, and formatting are unified, the product becomes more useful to both humans and future AI tooling.

Boundaries

Scope protects trust

The generator’s structure-driven limitation is a feature, not a weakness. It keeps the platform honest about what it is actually designed to do.

Why the platform starts with outpatient systems

Outpatient specialties offer repeatable workflows, high-value documentation patterns, and clear opportunities for systematization without trying to solve all of medicine at once.

Deliberate focus

MDS is intentionally focusing on clinic-based documentation systems because narrow, high-quality structure is a stronger foundation than broad but poorly bounded ambition.
