Generator architecture

A documentation generator built to organize input, not invent medicine.

The MDS generator is designed to transform user-provided clinical content into structured dot phrase output. It does not independently perform medical research, create missing follow-up guidance, or fabricate clinical recommendations that were never supplied.

That boundary is a core product advantage: it keeps the generator honest, scoped, and better aligned with trustworthy outpatient documentation workflows.

Generator boundary rule

If the user does not provide the content, the generator should not invent it.

Examples: no follow-up guidance without supplied follow-up details, no medication risk counseling without supplied counseling language, and no fabricated procedure details.

What the generator does — and does not do

The distinction matters for product clarity, legal and clinical scope, and future trust with users and EHR-facing workflows.

It does

Structured documentation work

  • Organize and standardize user-provided content
  • Convert content into documentation-ready dot phrase output
  • Apply family architecture, naming rules, placeholders, and formatting logic
  • Support controlled post-generation refinement requests
It does not

Research or invent

  • Perform independent medical research
  • Supply missing treatment plans, counseling, or specialty details
  • Do the user’s clinical thinking for them
  • Override the outpatient, structure-driven boundary of the platform

How the workflow is designed

The generator is built around a clean sequence: standard generation first, optional controlled refinement second.

1. Select family

The user chooses the documentation family or structured request type.

2. Provide content

The user supplies the clinical language, topic, and any required details for inclusion.

3. Generate output

The system returns normalized phrase output based on rules, examples, and supplied content.

4. Refine if needed

The user may request formatting or wording adjustments without creating new clinical content.

Standard mode vs batch mode

Batch mode is designed for large phrase builds, but it remains controlled and structured.

Why the generator matters to AI-assisted chart building

This is not generic free-form generation. It is a bounded output system aligned to outpatient documentation architecture.

When AI is grounded in clear structure, family rules, consistent formatting, and explicit content boundaries, it becomes easier to reason about quality, trace output behavior, and discuss future EHR implementation paths responsibly.
